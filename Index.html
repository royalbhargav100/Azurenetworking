Azure Networking Fundamentals

📘
ఏమిటి ఇది? (What is this?) Azure networking అంటే మీ cloud resources అన్నీ ఒకదానితో ఒకటి secure గా communicate చేసుకోవడానికి కావలసిన infrastructure. Real world లో ఒక building లో rooms, doors, locks, corridors ఉన్నట్టు — Azure లో VNet, Subnets, NSGs, Routes ఉంటాయి.
🌐
Azure Region
Physical datacenter location
మొత్తం Azure resources ఒక Region లో deploy అవుతాయి. Example: East US, West Europe, Central India. Banking లో డేటా sovereignty కోసం region selection చాలా important.
🏗️
Resource Group
Logical container
Related resources కి ఒక logical container. ఒక project లేదా environment అన్ని resources ఒక RG లో group చేయవచ్చు. Access control, billing, lifecycle management కోసం వాడతారు.
📡
Availability Zones
99.99% SLA
ఒక region లో 3 physically separated datacenters. Banking లో 99.99% uptime కావాలంటే resources zones అన్నింటిలో spread చేయాలి — Zone Redundant setup.
02

Virtual Network (VNet) — Deep Dive

🌐 VNet అంటే ఏమిటి? BEGINNER

VNet అనేది Azure లో మీ own private network. Real world లో మీ home/office network లాంటిది. మీరే IP address range define చేస్తారు, traffic ఎలా flow అవ్వాలో control చేస్తారు.

VNet Key Properties

VNet Properties: Address Space : 10.0.0.0/8 (16M IPs) Region : East US 2 Subnets : Multiple, non-overlapping DNS Servers : Custom or Azure default Peering : Cross-VNet communication Service Endpoints: PaaS access control DDoS Protection: Standard / Basic

Banking Production VNet Design

# Banking Grade — Hub VNet hub_vnet: 10.0.0.0/16 subnet_firewall : 10.0.0.0/26 subnet_bastion : 10.0.1.0/27 subnet_gateway : 10.0.2.0/27 subnet_management: 10.0.3.0/24 # Spoke VNet — Banking App spoke_banking: 10.1.0.0/16 subnet_app : 10.1.1.0/24 subnet_aks : 10.1.2.0/22 subnet_db : 10.1.6.0/24
🏦
Banking Best Practice: Hub-Spoke Topology ప్రతి workload (Banking, Payments, Analytics, DR) separate Spoke VNet లో isolate చేయాలి. Hub VNet లో Firewall, Bastion, VPN/ExpressRoute centralize చేయాలి. ఇది PCI-DSS, SOC2 compliance కోసం mandatory.

Subnets — IP Design & Segmentation

🧩 Subnet అంటే ఏమిటి? BEGINNER

VNet ని smaller segments గా divide చేస్తాం. ఒక building లో floors లాంటివి. ప్రతి subnet కి separate security rules apply చేయవచ్చు.

# Subnet CIDR Planning Rule: Azure reserves first 4 + last 1 IPs /28 = 16 IPs → 11 usable (small) /27 = 32 IPs → 27 usable (medium) /24 = 256 IPs → 251 usable (large) /22 = 1024 IPs→ 1019 usable(AKS) /16 = 65536 IPs (VNet)
🏦 Banking Grade Subnet Design ADVANCED
# Production Banking Subnets AzureFirewallSubnet 10.0.0.0/26 AzureBastionSubnet 10.0.1.0/27 GatewaySubnet 10.0.2.0/27 AppGatewaySubnet 10.0.3.0/24 snet-aks-system 10.1.0.0/24 snet-aks-user 10.1.4.0/22 snet-apim 10.1.8.0/27 snet-database 10.1.9.0/24 snet-privateendpoints 10.1.10.0/24
⚠️
Important: AzureFirewallSubnet & AzureBastionSubnet ఈ రెండు subnets కి exact names వాడాలి — Azure automatically identify చేస్తుంది. AzureFirewallSubnet minimum /26 అయి ఉండాలి. GatewaySubnet minimum /27.

Network Security Groups (NSG)

🛡️
NSG అంటే ఏమిటి? NSG అనేది Azure లో stateful firewall. Subnet లేదా NIC level లో attach చేయవచ్చు. Inbound + Outbound rules define చేస్తాం — priority based, lower number = higher priority.
Banking NSG Rules — AKS Node Subnet
# NSG: snet-aks-nodes — Banking Grade INBOUND RULES: Priority 100: Allow AzureLoadBalancer → Any * # Health probes Priority 110: Allow VirtualNetwork → Any * # Pod communication Priority 120: Allow 10.0.0.0/26 → Any 443 # From Firewall subnet Priority 200: Deny Internet → Any * # Block all internet Priority 65500: Deny * → Any * # Default deny all OUTBOUND RULES: Priority 100: Allow Any → AzureContainerRegistry 443 # Pull images Priority 110: Allow Any → AzureMonitor 443 # Telemetry Priority 120: Allow Any → VirtualNetwork * # Internal Priority 200: Deny Any → Internet * # Egress lockdown

Routing — UDR & Azure Firewall Forced Tunneling

🗺️
User Defined Routes (UDR)
Custom routing rules
Azure default routes ని override చేయడానికి UDR వాడతాం. Banking లో అన్ని internet traffic Azure Firewall ద్వారా force చేయడానికి UDR mandatory.
# Force All Egress via Firewall Route Name: force-to-firewall Address Prefix: 0.0.0.0/0 Next Hop Type: Virtual Appliance Next Hop IP: 10.0.0.4 # Firewall IP
🔥
Forced Tunneling
All traffic via Firewall
ప్రతి packet Azure Firewall ద్వారా inspect అవ్వాలి — ఇది banking compliance కోసం essential. UDR + Firewall combination తో East-West మరియు North-South traffic control చేస్తాం.
# Traffic Flow Banking Internet → AppGW+WAF → Firewall → AKS Ingress → Pods → Private Endpoints → DB AKS Egress → UDR → Firewall → ACR / KeyVault / Monitor

DNS — Private DNS Zones & Resolution

🔍 Private DNS Zones INTERMEDIATE

Private DNS zones అంటే VNet లో internal name resolution. Banking లో Private Endpoints కి private DNS zones mandatory — otherwise traffic public internet వైపు వెళ్ళే risk ఉంటుంది.

# Private DNS Zones — Banking privatelink.database.windows.net → SQL Server private endpoints privatelink.blob.core.windows.net → Storage Account private access privatelink.vaultcore.azure.net → Key Vault private access privatelink.azurecr.io → Container Registry private pull privatelink.servicebus.windows.net → Service Bus internal only
# DNS Resolution Flow AKS Pod queries: mydb.database.windows.net ↓ CoreDNS → Azure DNS (168.63.129.16) ↓ Private DNS Zone lookup ↓ Returns: 10.1.10.5 (Private IP!) ↓ Traffic stays INSIDE VNet ✅ # No public internet exposure!

VNet Peering & Global Peering

🔗
VNet Peering
Low latency, high bandwidth
రెండు VNets ని private Microsoft backbone network ద్వారా connect చేస్తుంది. Public internet ఉపయోగించదు. Latency చాలా తక్కువ — same region లో sub-millisecond.
Non-transitive Bidirectional Low latency No downtime
🌍
Global VNet Peering
Cross-region connectivity
Different Azure regions మధ్య VNet peering. Banking లో DR (Disaster Recovery) setup కోసం — Primary region East US 2 ↔ Secondary region West Europe. Data replication తక్కువ latency తో.
⚠️
Peering non-transitive! Hub-Spoke లో hub ద్వారా routing కావాలంటే Azure Firewall లేదా NVA కావాలి.

Azure Firewall — Premium (Banking Grade)

🔥
Azure Firewall Premium — Banking Mandatory Features TLS inspection, IDPS (Intrusion Detection Prevention), URL filtering, threat intelligence. Banking లో Premium SKU మాత్రమే use చేయాలి — Standard SKU insufficient for PCI-DSS.
# Azure Firewall Premium — Key Rules Application Rules (FQDN based): Allow *.microsoft.com port 443 Allow *.azure.com port 443 Allow *.ubuntu.com port 80,443 Deny * port * Network Rules (IP/Port based): Allow 10.1.0.0/16 → 10.2.0.0/16 * Deny * → Internet * IDPS Mode: Alert and Deny TLS Inspection: Enabled (own CA cert) Threat Intel: Alert and Deny
🛡️ Threat Intelligence Microsoft threat feeds — auto block malicious IPs/domains
🔬 IDPS — Intrusion Detection Signature-based attack detection, 58,000+ signatures
🔒 TLS Inspection Decrypt → inspect → re-encrypt HTTPS traffic
🌐 URL Filtering Category-based web filtering (gambling, social, malware)
📊 Policy + Firewall Manager Centralized policy across all Hub-Spoke firewalls

VPN Gateway & ExpressRoute

🔐
VPN Gateway
IPsec/IKE encrypted tunnel
Public internet ద్వారా encrypted connection. Branch offices, remote workers కోసం. Banking లో critical data కి ExpressRoute prefer చేస్తారు.
SKU: VpnGw2AZ (Zone redundant) Type: Route-based Throughput: 1.25 Gbps Connections: S2S + P2S IKE: v2 (AES-256, SHA-256)
⚡
ExpressRoute
Private dedicated connection
Internet bypass — dedicated fiber connection. Banking లో on-premises data center to Azure. Latency 1-5ms, bandwidth upto 100Gbps. No internet exposure — regulatory compliance perfect.
Circuit SKU: Premium Bandwidth: 1 Gbps - 100 Gbps Peering: Private + Microsoft Redundancy: Dual circuits (BGP) SLA: 99.95%
🏦
Banking Recommendation: ExpressRoute + VPN (Active-Active) Primary: ExpressRoute (1Gbps). Failover: VPN Gateway. ఇలా చేస్తే connectivity never goes down. RBI mandates dedicated connectivity for core banking systems.
